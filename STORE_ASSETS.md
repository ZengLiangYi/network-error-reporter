# Store Assets Checklist

## Required

- `icons/icon128.png`
- At least 1 screenshot of the extension in use

## Recommended screenshots

### Screenshot 1

Title:
`Generate reports from failed requests`

Content:

- show the `Error Report` panel inside DevTools
- keep the left request list visible
- keep the right report preview visible
- use a realistic failed `Fetch/XHR` request

### Screenshot 2

Title:
`Readable report preview, not raw DevTools noise`

Content:

- focus on the report preview area
- show extracted headers, request payload, and response summary
- avoid showing sensitive real tokens

### Screenshot 3

Title:
`Export as Markdown or image`

Content:

- show the preview toolbar with `复制 Markdown` and `导出图片`
- optionally show a copied/exported result in chat or note-taking context

## Screenshot guidance

- use real extension UI, not mockups
- keep request data anonymized
- avoid showing private domains, tokens, cookies, or user data
- keep the DevTools frame visible so users understand this is a DevTools extension
